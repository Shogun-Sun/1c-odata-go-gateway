# Academic Booking API

Сервис для управления бронированием аудиторий учебного заведения. API выступает в роли связующего звена (middleware) между пользовательскими приложениями и системой 1С:Предприятие, обеспечивая работу с расписанием и занятостью помещений через протокол OData.

## Основные возможности

* **Управление бронированием:** Автоматизация процессов бронирования аудиторий.
* **Интеграция с 1С:** Безопасное проксирование данных из учетной системы 1С.
* **Контейнеризация:** Готовая к запуску инфраструктура в Docker (1С-сервер + API-сервис на Go).

## Начало работы

### Установка и запуск

1. **Склонируйте репозиторий:**
```bash
git clone https://github.com/Shogun-Sun/1c-odata-go-gateway.git
cd academic-booking-api
```

Или

```bash
git clone git@github.com:Shogun-Sun/1c-odata-go-gateway.git
cd academic-booking-api
```

2. **Настройка окружения:**
Скопируйте пример файла конфигурации и заполните данные для подключения:
```bash
cp .env.example .env
```

3. **Запуск:**
Соберите и запустите проект командой:
```bash
docker compose up -d --build
```


## Резервное копирование

Проект поддерживает автоматическое создание резервных копий (дампов) файловой базы данных 1С в формате `.dt`. Из-за специфики работы учебной версии платформы 1С (требование монопольного доступа и наличие графического окружения), резервное копирование вынесено на **хостовую машину** и автоматизировано через системный планировщик `cron`.

Скрипт бэкапа автоматически останавливает контейнер веб-сервера 1С для снятия блокировок, очищает остаточные `.lck` файлы, выполняет выгрузку базы с помощью Конфигуратора 1С и запускает веб-сервер обратно.

### Настройка и управление

Перед активацией планировщика необходимо один раз проверить и скорректировать пути внутри скриптов.

1. **Настройка параметров:**
Откройте ```backup_academic_booking.sh``` на хосте и убедитесь, что переменные (особенно пути к проекту, параметры авторизации 1С) указаны верно для вашей системы.

2. **Сделайте скрипт управления исполняемым:**
```bash
sudo chmod +x manage_cron.sh
```

3. **Включите автоматические бэкапы:**
```bash
sudo ./manage_cron.sh on
```

*Скрипт автоматически пропишет абсолютный путь к бэкаперу и добавит задачу в системный `crontab` на ежедневный запуск в 23:01.*
4. **Отключите автоматические бэкапы (при необходимости):**
```bash
sudo ./manage_cron.sh off
```

*Скрипт удалит задачу бэкапа.*
5. **Проверить текущее расписание:**
```bash
crontab -l
```

### Структура и ротация

* Все готовые резервные копии и логи их выполнения складываются в папку `/home/user_name/backups/`, которй можно изменить в скрипте.
* Формат именования файлов: `base1__YYYY-MM-DD-HH-MM-SS.dt` и аналогичный `.log`.
* Скрипт бэкапа имеет встроенную функцию **ротации**: по умолчанию хранятся файлы только за последние **2 дня**, более старые дампы удаляются автоматически для экономии места на диске.
